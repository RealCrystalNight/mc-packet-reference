implementation Implementation Cases

C00PacketKeepAlive is the client"s response to the server"s S00PacketKeepAlive. The server periodically sends a keepalive request with a random integer ID, and the client MUST echo that ID back within ~30 seconds or the server disconnects with "Timed out". This packet is one of the most commonly intercepted in 1.8.9 clients because it directly controls how the anti-cheat measures connection latency and packet timing. Intercepting or delaying it is the foundation of PingSpoof, many Disabler modes, and Timer/LagSwitch modules.

PingSpoof (Astro 2.2.1, Sigma 3.9, Sigma 2.08, Tenacity 5.1, Lycoris 2.0, November 0.2, Raze Recode 1.1)

Fakes higher ping by delaying the keepalive response. Useful for looking like you have lag (to explain away impossible angles/fast reactions) or for anti-cheat bypass.

When S00PacketKeepAlive arrives (incoming), the module cancels the normal handler and saves the keepalive ID. Then instead of immediately sending C00 back, it waits a configurable delay (e.g. 500ms - 3000ms). After the timer fires, it sends C00PacketKeepAlive with the stored ID back. Some clients modify the ID by adding a random offset to make it look "natural".

// Sigma 3.9 - PingSpoof.java (queue-based delay, ~60 lines) // Reproduced from actual source public class PingSpoof extends Module { private ArrayList<Packet> packetList = new ArrayList<>(); private Timer timer = new Timer(); @EventTarget public void onPacket(EventPacket ep) { // On outgoing: queue C00PacketKeepAlive, cancel immediate send if (ep.isOutgoing() && ep.getPacket() instanceof C00PacketKeepAlive && mc.thePlayer.isEntityAlive()) { packetList.add(ep.getPacket()); event.setCancelled(true); return; } // On incoming: capture keepalive ID for response if (ep.isIncoming() && ep.getPacket() instanceof S00PacketKeepAlive) { S00PacketKeepAlive s00 = (S00PacketKeepAlive) ep.getPacket(); lastKey = s00.func_149134_c(); // store the key } } @EventTarget public void onTick(EventTick event) { // Release queued packets after configured delay if (timer.delay(1000 * waitSeconds) && !packetList.isEmpty()) { for (Packet p : packetList) { mc.thePlayer.sendQueue.getNetworkManager() .sendPacketNoEvent(p); } // Also send a fake keepalive with large key to confuse anticheat mc.thePlayer.sendQueue.getNetworkManager() .sendPacketNoEvent(new C00PacketKeepAlive(10000)); packetList.clear(); } } }
Vanilla hook: NetHandlerPlayClient.handleKeepAlive() — the method that receives S00 and sends back C00. Override via EventReceivePacket before this method runs.
Vanilla timeout is ~30 seconds. Most anti-cheats (Watchdog, AAC, NCP) check that the lag is consistent — if you delay exactly 500ms every time, they flag it. Smart PingSpoof modules randomize the delay by ±15%. Some anti-cheats also check that the keepalive ID matches what they sent; random modifications to the ID can trigger flags.
Disabler — Watchdog (Hypixel) mode (Tenacity 5.1, Tenacity 4.0, Lycoris 2.0, November 0.2)

Hypixel"s Watchdog anti-cheat uses keepalive timing + transaction (C0F) ordering to detect cheat modules. By queueing C00+C0F together and releasing them after a precise delay, the module "disables" Watchdog"s ability to correlate movement vs server position.

The disabler intercepts BOTH C00PacketKeepAlive and C0FPacketConfirmTransaction. For the first 350ms of a movement burst, all C00 and C0F packets are cancelled and saved to a queue. After 350ms, ALL queued packets are flushed at once. This creates a burst of "valid" transactions that saturate Watchdog"s buffer, causing it to stop checking movement legitimacy.

// Tenacity 5.1 - Disabler.java (Watchdog mode, lines 145-165) // Reproduced from actual source: https://github.com/TenacityMC/Tenacity @Subscribe public void onPacketSend(PacketEvent event) { if (mc.thePlayer == null) return; Packet<?> packet = event.getPacket(); String mode = disablerMode.getSelectedOption(); if (mode.equals("Watchdog")) { // Queue C0FPacketConfirmTransaction + C00PacketKeepAlive for 350ms // Watchdog uses a sliding window of ~500ms to correlate client actions. // By withholding all C0F and C00 for 350ms and then flushing them, // Watchdog's window desyncs and stops flagging movement. if (!timer.hasTimeElapsed(350)) { if (packet instanceof C0FPacketConfirmTransaction || packet instanceof C00PacketKeepAlive) { event.cancel(); watchdogC0FC00Packets.add(packet); } } else if (!watchdogC0FC00Packets.isEmpty()) { watchdogC0FC00Packets.forEach(PacketUtils::sendPacketNoEvent); watchdogC0FC00Packets.clear(); } } else if (mode.equals("Verus")) { // Verus: cancel ALL transaction + keepalive if (packet instanceof C0FPacketConfirmTransaction || packet instanceof C00PacketKeepAlive) { event.cancel(); } } else if (mode.equals("MMC")) { // Minemen Club: cancel C0F+C00, store in separate queue if (packet instanceof C0FPacketConfirmTransaction || packet instanceof C00PacketKeepAlive) { event.cancel(); packets.add(packet); } } }
Vanilla hook: The keepalive ID flows through: NetworkManager.sendPacket() → EventSendPacket → Netty pipeline. Cancelling in EventSendPacket is the cleanest approach.
Watchdog: this technique (queue + burst) works because Watchdog uses a sliding window of ~500ms. By withholding all data for 350ms and then sending it all at once, the server-side check sees a "snapshot" of valid transactions. AAC: queues keepalive and transaction together, drops every Nth one.
Disabler — Faithful / Erisium / Mineplex mode (Sigma 3.9, Memeware 7.3, Lycoris 2.0)

Cancels C00 entirely or replaces it with a corruted key to confuse specific anti-cheats.

Faithful: cancel all C00 outgoing packets. The server never receives a response, but other tricks (like movement desync) keep the connection alive. Erisium: cancel and re-send with a random key. Mineplex: modify the key value by subtracting a random amount each time, making the keepalive look like it"s drifting.

// Memeware 7.3 - Disabler.java (Mineplex + Faithful modes) // Reproduced from actual source // Mineplex: randomly shift keepalive key to bypass anticheat if (isMode("Mineplex Combat")) { if (event.getPacket() instanceof C00PacketKeepAlive) { C00PacketKeepAlive packetKeepAlive = (C00PacketKeepAlive) event.getPacket(); // Subtract random 3-127 from the key value. // The server expects a specific key but Mineplex's anticheat // accepts a range around the expected value. packetKeepAlive.key -= RandomUtils.nextInt(3, 128); } } // Faithful: shift keepalive key by exactly 31 if (isMode("Faithful Movement")) { if (mc.thePlayer != null && event.getPacket() instanceof C00PacketKeepAlive) { C00PacketKeepAlive packetKeepAlive = (C00PacketKeepAlive) event.getPacket(); packetKeepAlive.key -= 31; } }
Vanilla hook: Same pattern — EventSendPacket. The key field on C00PacketKeepAlive can be modified via reflection or direct field access in MCP (the field is literally named "key" in 1.8.9).
Faithful: cancel all keepalives and rely on movement syncing. Works on servers that only check keepalive for timeout (not anticheat). Mineplex: shifting the key exploits a bug in Mineplex"s anticheat where it accepts a range of values around the expected key. Modern anti-cheats expect an exact match.
DialUpPing / LagSwitch (Jigsaw 0.24)

Causes deliberate lag by dropping keepalive packets. Makes the player appear "laggy" to avoid suspicion or triggers a temporary server freeze.

Every Nth keepalive packet is dropped entirely. The server eventually sees a timeout disconnect, but if the module starts sending again before the 30-second timeout, the connection survives. The player appears to have high packet loss.

// Jigsaw 0.24 - DialUpPing.java if (packet instanceof C00PacketKeepAlive) { if (tries != 0) { packet.cancel(); } // drop this keepalive tries++; if (tries > 149) tries = 0; }
Vanilla hook: EventSendPacket — same as PingSpoof.
Easy to detect server-side by checking packet loss ratio. Most ghost clients don"t use this; it"s a free/leisure client feature.
PingSpoof (Timestamp Queue) (November 0.2)

Uses a TreeMap with timestamp keys to queue and release C0F+C00 with precise timing.

Packets are stored in a TreeMap where the key is currentTimeMillis + delay. A tick handler periodically drains packets whose timestamp has passed.

// November 0.2 - PingSpoof.java (TreeMap queue) // Reproduced from actual source private final TreeMap<Long, Packet> packets = new TreeMap<>(); @EventTarget public void onSend(EventSendPacket event) { Packet p = event.getPacket(); if (p instanceof C0FPacketConfirmTransaction || p instanceof C00PacketKeepAlive) { // Schedule for future delivery long releaseAt = System.currentTimeMillis() + (long) this.ping.getValue(); this.packets.put(releaseAt, p); event.setCancelled(true); } } @EventTarget public void onTick(EventTick event) { // Release all packets whose time has come long now = System.currentTimeMillis(); while (!this.packets.isEmpty() && this.packets.firstKey() <= now) { Packet p = this.packets.pollFirstEntry().getValue(); this.sendPacket(p); } }
Vanilla hook: EventSendPacket — timestamp-based queue for both C0F and C00.
There are THREE primary ways to intercept this packet in 1.8.9 clients: 1. EventSendPacket/EventReceivePacket — Most common. The client has a mixin or edited NetworkManager that fires events before/after every packet. You check instanceof C00PacketKeepAlive or S00PacketKeepAlive. 2. NetHandlerPlayClient.handleKeepAlive() — Override the specific handler method. Only works for incoming S00, but gives you direct access. 3. NetworkManager.sendPacket() — Modify the sendPacket method to filter or queue C00 packets globally. This is how Blink and similar modules often work at the lowest level.
Astro 2.2.1: Uses EventReceivePacket, modifies the keepalive ID by adding mc.getDebugFPS()/2 as offset. Sigma 3.9: Full queue-based PingSpoof with configurable waitSeconds. Also has Disabler mode that completely cancels C00. Tenacity 5.1: Integrates C00 handling into the Watchdog disabler — queues C00 alongside C0F, flushes after 350ms. Lycoris 2.0: Sends a completely spoofed C00 with Integer.MIN_VALUE key for Faithful bypass. November 0.2: Uses a timestamp-indexed queue (TreeMap with System.currentTimeMillis() + delay as key). Memeware 7.3: Direct field mutation on the packet object (packetKeepAlive.key -= random bits). Raze v1.5b: Drops C00 entirely as part of a hypixel fast disabler (allows only every Nth C03). Jigsaw 0.24: Every 150th keepalive is dropped to simulate dial-up ping.
