Notes

implementation Implementation Cases

C07PacketPlayerDigging handles all block-breaking actions and item-use release. It has six actions: START_DESTROY_BLOCK (begin mining), ABORT_DESTROY_BLOCK (cancel mining), STOP_DESTROY_BLOCK (finish mining — block breaks), DROP_ALL_ITEMS (drop entire stack with Q), DROP_ITEM (drop one item), and RELEASE_USE_ITEM (release right-click — used for bows, eating, blocking). The first three are used by Nuker/SpeedMine/Fucker/BedFucker to destroy blocks. RELEASE_USE_ITEM is arguably more important — it"s the backbone of autoblock (resetting the sword after attack), NoSlowdown (cancelling the use-item slowdown), FastBow (instantly shooting), and FastUse/Eat.

Autoblock (Block-hitting / Sword blocking reset) (Slice v15, Mint B1, Gugustus, Koks v1)

Resets the block animation during combat to reduce damage taken and maintain blocking. Also triggers "block-hitting" which deals more knockback.

After every C02PacketUseEntity ATTACK, the module sends C07 RELEASE_USE_ITEM (same as pressing Q on a sword to stop blocking). This breaks the block animation early, causing the server to think the player stopped blocking momentarily. When combined with an immediate C08 (start blocking again), the server sees: attack → unblock → reblock → attack → unblock → reblock... which produces the block-hitting effect.

// Slice v15 - KillAura.java (autoblock) thePlayersendQueue.addToSendQueue(new C02PacketUseEntity(e, C02PacketUseEntity.Action.ATTACK)); thePlayersendQueue.addToSendQueue(new C07PacketPlayerDigging( C07PacketPlayerDigging.Action.RELEASE_USE_ITEM, new BlockPos(0, 0, 0), EnumFacing.DOWN)); // Mint B1 - KillAura.java mc.thePlayer.sendQueue.addToSendQueue(new C02PacketUseEntity(target, C02PacketUseEntity.Action.ATTACK)); mc.thePlayer.sendQueue.addToSendQueue( new C07PacketPlayerDigging(C07PacketPlayerDigging.Action.RELEASE_USE_ITEM, BlockPos.ORIGIN, EnumFacing.DOWN)); // Gugustus - KillAura.java mc.thePlayer.sendQueue.addToSendQueue(new C02PacketUseEntity(entity, Action.ATTACK)); mc.thePlayer.sendQueue.addToSendQueue( new C07PacketPlayerDigging(C07PacketPlayerDigging.Action.RELEASE_USE_ITEM, mc.thePlayer.getPosition(), EnumFacing.DOWN));
Vanilla hook: EventSendPacket — sent immediately after C02 ATTACK. The packet position doesn"t matter for RELEASE_USE_ITEM (BlockPos.ORIGIN / new BlockPos(-1,-1,-1) are both commonly used).
NoSlowdown (Item use speed) (Slice v15, Mint B1, Gugustus, Tenacity 6.0, November 0.2)

Prevents the slowness effect when using items (eating, blocking, bow charging) so the player can run at full speed while blocking.

While the player is using an item (e.g., blocking with sword), the module sends C07 RELEASE_USE_ITEM every movement tick. This tells the server "I stopped using the item", which cancels the slow effect. The client keeps the item-use animation visually. On the next tick, a C08 places an invalid block (autoblock start) to resume the animation. The net effect: the player appears to be blocking/using item but moves at full speed.

// Slice v15 - NoSlowDown.java thePlayersendQueue.addToSendQueue( new C07PacketPlayerDigging(C07PacketPlayerDigging.Action.RELEASE_USE_ITEM, new BlockPos(0, 0, 0), EnumFacing.DOWN)); // Mint B1 - NoSlow.java mc.thePlayer.sendQueue.addToSendQueue( new C07PacketPlayerDigging(C07PacketPlayerDigging.Action.RELEASE_USE_ITEM, BlockPos.ORIGIN, EnumFacing.DOWN)); // November 0.2 - NoSlow.java (EventMotion based) if (isBlocking() && shouldNoSlow) { mc.thePlayer.sendQueue.addToSendQueue( new C07PacketPlayerDigging(C07PacketPlayerDigging.Action.RELEASE_USE_ITEM, BlockPos.ORIGIN, EnumFacing.DOWN)); }
Vanilla hook: EventSendPacket / EventMotion (PRE/POST). The C07 must be sent during the movement packet phase.
FastBow / FastUse / FastEat (Gugustus, Slice v15, November 0.2)

Fully charges a bow instantly (FastBow) or consumes items faster than normal (FastUse).

The bow charging is controlled by the server — the bow fires when RELEASE_USE_ITEM is received. Normally this only happens when the player releases right-click after charging. FastBow sends both C07 RELEASE_USE_ITEM and C08 place back-to-back rapidly, tricking the server into thinking the bow was fully charged. For eating, FastUse sends RELEASE_USE_ITEM + place repeatedly to accelerate the consumption animation.

// Gugustus - FastBow.java mc.getNetHandler().addToSendQueue( new C07PacketPlayerDigging(C07PacketPlayerDigging.Action.RELEASE_USE_ITEM, BlockPos.ORIGIN, EnumFacing.DOWN)); // Slice v15 - FastUse.java // Rapidly toggle item usage mc.thePlayer.sendQueue.addToSendQueue( new C07PacketPlayerDigging(C07PacketPlayerDigging.Action.RELEASE_USE_ITEM, new BlockPos(-1,-1,-1), EnumFacing.DOWN)); // Full FastBow sequence: mc.getNetHandler().addToSendQueue( new C07PacketPlayerDigging(C07PacketPlayerDigging.Action.RELEASE_USE_ITEM, BlockPos.ORIGIN, EnumFacing.DOWN)); mc.thePlayer.sendQueue.addToSendQueue( new C08PacketPlayerBlockPlacement(new BlockPos(-1, -1, -1), 255, mc.thePlayer.getHeldItem(), 0, 0, 0));
Vanilla hook: NetHandlerPlayServer.processPlayerDigging() — the server side. On the client, hook into EventSendPacket or use a timer-driven packet scheduler.
Nuker / BedFucker (Block destruction) (Slice v15, Gugustus, Koks v1, Swift Developer Alpha)

Destroys blocks rapidly — either the block you"re looking at (bedfuck) or all blocks in a radius (nuker).

The module sends C07 START_DESTROY_BLOCK followed immediately by STOP_DESTROY_BLOCK for each target block. The server processes the start → cancel → start → cancel pattern and breaks the block instantly (because the client says it finished mining before the server can recalculate). For bed/block destruction, the module iterates through nearby blocks and sends START+STOP for each one.

// Slice v15 - BedFucker.java thePlayersendQueue.addToSendQueue( new C07PacketPlayerDigging(C07PacketPlayerDigging.Action.START_DESTROY_BLOCK, pos, EnumFacing.UP)); thePlayersendQueue.addToSendQueue( new C07PacketPlayerDigging(C07PacketPlayerDigging.Action.STOP_DESTROY_BLOCK, pos, EnumFacing.UP)); // Gugustus - Fucker.java (same pattern) mc.thePlayer.sendQueue.addToSendQueue( new C07PacketPlayerDigging(C07PacketPlayerDigging.Action.START_DESTROY_BLOCK, blockPos, enumFacing)); mc.thePlayer.sendQueue.addToSendQueue( new C07PacketPlayerDigging(C07PacketPlayerDigging.Action.STOP_DESTROY_BLOCK, blockPos, enumFacing)); // Koks v1 - BedFucker.java mc.thePlayer.sendQueue.addToSendQueue( new C07PacketPlayerDigging(C07PacketPlayerDigging.Action.START_DESTROY_BLOCK, this.blockPos, EnumFacing.DOWN)); mc.thePlayer.sendQueue.addToSendQueue( new C07PacketPlayerDigging(C07PacketPlayerDigging.Action.STOP_DESTROY_BLOCK, this.blockPos, EnumFacing.DOWN));
Vanilla hook: NetHandlerPlayServer.processPlayerDigging() server-side. Client-side: send START+STOP in immediate succession.
SpeedMine / FastBreak (Swift Developer Alpha, Tenacity 6.0)

Breaks blocks faster than normally possible by manipulating the dig cycle.

Intercepts C07 START_DESTROY_BLOCK packets and modifies the timing. Some modules also resend the block break animation packets at an accelerated rate. The server sees the client claiming to have progressed through the mining stages faster than the block"s hardness allows.

// Swift Developer Alpha - FastBreak.java if (event.getPacket() instanceof C07PacketPlayerDigging) { C07PacketPlayerDigging packet = (C07PacketPlayerDigging) event.getPacket(); if (packet.getStatus() == C07PacketPlayerDigging.Action.START_DESTROY_BLOCK) { // Modify block break timing mc.getNetHandler().addToSendQueue( new C07PacketPlayerDigging(C07PacketPlayerDigging.Action.STOP_DESTROY_BLOCK, packet.getPosition(), packet.getFacing())); } }
Vanilla hook: EventSendPacket — detect START_DESTROY_BLOCK, immediately precede with STOP_DESTROY_BLOCK.
AutoSoup / AutoPot (DROP_ITEM) (Mint B1)

Quickly drops empty soup bowls after consumption.

After eating soup, sends DROP_ITEM action to discard the empty bowl from inventory. The packet position is irrelevant for DROP_ITEM/DROP_ALL_ITEMS.

// Mint B1 - AutoPot.java mc.thePlayer.sendQueue.addToSendQueue( new C07PacketPlayerDigging(C07PacketPlayerDigging.Action.DROP_ITEM, BlockPos.ORIGIN, EnumFacing.DOWN));
Vanilla hook: Same as other C07 actions.
All uses go through EventSendPacket. The action enum determines behavior. RELEASE_USE_ITEM is the most versatile action (used by autoblock, noslow, fastbow, fasteat). START_DESTROY_BLOCK + STOP_DESTROY_BLOCK are always paired for block-breaking modules. DROP_ITEM is rarely used by modules (most use the vanilla drop stack key instead). Server handler: NetHandlerPlayServer.processPlayerDigging() dispatches based on action. RELEASE_USE_ITEM maps to the current item"s onItemRightClick release logic. The block position field is only meaningful for DESTROY_BLOCK actions — RELEASE_USE_ITEM ignores it.
Slice v15: BedFucker uses START+STOP. NoSlow uses RELEASE. FastUse uses RELEASE. Autoblock uses RELEASE after ATTACK. Mint B1: Autoblock + NoSlow + AutoPot + Disabler (START spam) + Fly (RELEASE for speed). Gugustus: Fucker (START+STOP), NoSlow/NoSlow2 (RELEASE + DROP combos), Autoblock, FastBow. Koks v1: BedFucker (START+STOP), Autoblock (RELEASE). Swift Developer Alpha: Nuker (START+STOP), FastBreak (intercept and modify). November 0.2: NoSlow (EventMotion RELEASE).
