Notes

implementation Implementation Cases

C0EPacketClickWindow is the most complex inventory packet — 6 fields controlling item movement. Used by ALL inventory modules (ChestStealer, AutoArmor, InvManager, AutoPot). Rise 6.0 BufferAbuseInventoryMove exploits it by sending empty C0E() packets to desync server inventory buffer (reduces anti-cheat checks). RektSky B5 ChestStealer constructs C0E directly with custom transaction IDs for silent stealing.

BufferAbuseInventoryMove (Rise 6.0) (Rise 6.0)

Abuse C0EPacketClickWindow buffer desync to reduce anti-cheat checks on inventory actions. Sends burst of empty C0E() packets to overflow the server's transaction tracking buffer.

When the player has clicked enough inventory slots (modulo clicksSetting), the module sends N empty C0EPacketClickWindow() packets in a single pre-motion. The server's C0E tracker overflows and stops validating subsequent inventory clicks. All queued legitimate C0E packets are flushed after the burst.

Vanilla hook: NetHandlerPlayServer.processClickWindow(C0EPacketClickWindow) — re-executes slotClick server-side with same parameters. Validates actionNumber against container transaction counter. If mismatched, sends S32PacketConfirmTransaction(accepted=false) and reverts client inventory via S30PacketWindowItems.
BufferAbuse overflows the transaction ID counter. The server tracks actionNumber as a short (max 32767). Sending 10+ C0E in one tick causes quick overflow. Anti-cheats detect this by monitoring transaction sequence gaps — Grim flags C0E packets with non-sequential action numbers.
ChestStealer Silent Click (RektSky B5) (RektSky B5)

Silent chest stealing that constructs C0EPacketClickWindow directly (bypassing PlayerControllerMP) to avoid client-side sound effects and visual updates. The server processes the click but the client doesn't show the item movement.

The module constructs C0E with the correct windowId, slotIndex, mode=1 (shift-click), and a custom transaction ID from the container. It then calls inventory.slotClick() locally to sync client-side state without GUI sound. The packet is sent silently via addToSendQueue.
