Notes

Reason 0: bed can't be set (respawn). Reason 1/2: rain toggle. Reason 3: gamemode change (value=0-3). Reason 4: credits. Reason 5: demo. Reason 6: arrow hitting entity. Reason 7/8: fade values. Reason 9: elder guardian effect.

implementation Implementation Cases

S2BPacketChangeGameState controls global state via reason+value pairs. NoRain modules cancel reasons 1/2 (rain start/stop) to keep weather clear. Gamemode detection uses reason 3 to track creative/survival switches. Disabler modules read reason 10 (end combat) indirectly through this packet.

NoRain / NoWeather (Gugustus / Tenacity 6.0) (Gugustus, Tenacity 6.0, Diablo 2.1)

Cancel S2BPacketChangeGameState when reason=1 (start rain) or reason=2 (stop rain) to keep weather permanently clear. The server sends rain updates but the client ignores them.

On EventReceivePacket, check instanceof S2BPacketChangeGameState. If getGameState() is 1 or 2, cancel the packet. The client's rain strength stays at 0.
