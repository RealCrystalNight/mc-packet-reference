Notes

Sent immediately after C00Handshake when the nextState is STATUS (1). The server responds with S00PacketServerInfo (JSON status). Used by the multiplayer server list screen (OldServerPinger.java).

implementation Implementation Cases

C00PacketServerQuery is the empty request packet sent during the STATUS protocol state. After sending C00Handshake with requestedState=STATUS, the client sends this to ask the server for its status (MOTD, player count, version, favicon). The server responds with S00PacketServerInfo containing a JSON string. While rarely targetted directly by modules, it's integral to the server pinging system and is used by several disabler modes in Aqua 13 that manipulate the status request/response cycle to bypass anti-cheat state resets during gameplay.

Server List Pinger (Vanilla Flow) (Rise 6.0, Jigsaw 0.26, RektSky B5, Sigma 4.11, Aqua 13, Memeware 7.3)

The standard Minecraft server list uses this packet to fetch a server's MOTD, player count, favicon, and protocol version before the player joins. Every client has this in OldServerPinger.java.

OldServerPinger opens a short TCP connection, sends C00Handshake(state=STATUS), immediately sends C00PacketServerQuery, waits for S00PacketServerInfo, then sends C01PacketPing to measure latency, receives S01PacketPong, and closes the connection. The entire STATUS exchange happens on a separate connection from the gameplay connection — it's a one-shot request/response.

Disabler (Status Request Hook, Aqua 13) (Aqua 13)

The Aqua 13 Disabler imports C00PacketServerQuery as part of its packet manipulation arsenal. It hooks into the STATUS state to detect when the client is pinging vs playing, and adjusts its disabler behavior accordingly.

Aqua 13's Disabler imports both C00PacketServerQuery and C01PacketPing (status packets) alongside play packets. This suggests the disabler monitors ALL packet types, not just PLAY state packets. When the client is in STATUS state (server pinging), the disabler may behave differently than when in PLAY state (gameplay), avoiding state desyncs that could trigger kicks.

Importing status packets in the Disabler suggests awareness of connection state. If a disabler tries to manipulate PLAY packets while still in STATUS state, it could cause the client to get stuck in the wrong protocol state. Anti-cheats monitor for state mismatches.
Client-side: OldServerPinger.java → networkmanager.sendPacket(new C00PacketServerQuery()). The empty packet triggers the server's NetHandlerStatusServer to respond with S00PacketServerInfo. Server-side: NetHandlerStatusServer.processServerQuery(C00PacketServerQuery) generates the JSON status response. The response includes: version (name + protocol), players (max + online + sample), description (MOTD chat component), favicon (base64 PNG), and Forge mod list (if applicable). Note: This packet is only valid in STATUS state. Sending it in PLAY state would cause the server to disconnect with 'Received packet for wrong protocol state'.
