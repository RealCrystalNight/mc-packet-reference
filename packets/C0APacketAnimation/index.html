0x0A C0APacketAnimation

Player swings their arm (left click or server-side swing). Empty packet with no payload.

Notes

When the server receives this, it broadcasts S0BPacketAnimation to nearby players. The empty packet just notifies the server of the swing. Client-side swings (swingItem()) are purely local and don't send this packet.

implementation Implementation Cases

C0APacketAnimation notifies the server that the player swung their arm. Server broadcasts S0BPacketAnimation to nearby players. Used by KillAura (swing before/after attacks), Scaffold (swing on block place), NoSwing modules (cancel to hide swing), and AutoBlock (swing as part of block-hit cycle). Also used by BadPacketsComponent (Rise) to track attack state, and PingSpoofComponent to delay swing packets.

NoSwing / Animation Control (Rise 6.0 KillAura) (Rise 6.0, Sigma 4.11, RektSky B5)

Control when C0APacketAnimation is sent during combat — either send it for every attack, send silently (packet-only without local animation), or disable entirely. NoSwing cancels C0A to hide arm swing from other players while still dealing damage.

Rise 6.0 KillAura sends C0APacketAnimation as part of the attack sequence via PacketUtil.send(). The module's PacketSendEvent listener detects outgoing C0A packets to track swing rate. BadPacketsComponent tracks that swings occur before attacks. NoSwing variants cancel this packet entirely.

Scaffold Swing (Rise 6.0 / Sigma 4.11) (Rise 6.0, Sigma 4.11, RektSky B5)

Send C0APacketAnimation after placing a block with Scaffold to appear legitimate.

After each block placement, the Module sends C0APacketAnimation so other players see the swing animation. This is the server-side equivalent of swingItem() — without it, the block would appear to place itself.

FreeCam Cancel (Rise 6.0) (Rise 6.0, Sigma 4.11)

Cancel C0APacketAnimation while in FreeCam to prevent interaction with the world. When the camera is detached from the body, any swings should be from the real body position, not the camera.

While FreeCam is active, all outgoing C0A packets are cancelled. This prevents the player from swinging at entities from the camera position which would reveal the FreeCam usage.
