Notes

S03PacketPlayer is the abstract base. C04/C05/C06 each have their own packet ID. The client sends C04 when only position changes, C05 when only rotation changes, and C06 when both change. onGround is ALWAYS present.

implementation Implementation Cases

C03PacketPlayer (and its subclasses C04PacketPlayerPosition, C05PacketPlayerLook, C06PacketPlayerPosLook) is THE most manipulated packet in all of 1.8.9 client development. It carries the player"s position, rotation, and on-ground state. The server uses it to update the player"s location and check for movement cheats. Flight, Speed, NoFall, Blink, Criticals, Damage, Phase, Step, AntiVoid, NoVoid, HighJump, Regen — all of them work by sending modified C03 packets. This single packet class is responsible for more anticheat flags than everything else combined.

NoFall (Gugustus, Mint B1, Koks v1, Tenacity 6.0, November 0.2)

Prevents fall damage by spoofing the onGround flag or sending ground packets mid-air.

Two main approaches. Approach 1 (simplest): Intercept C03/C04 in EventSendPacket and setOnGround(true) — the server sees you as on ground even if you"re falling. Approach 2 (better for bypass): Send a fake C04 with onGround=true at a position slightly below you right before you hit the ground, confusing the server"s fall distance calculation.

// Gugustus - NoFall.java (OnGround + NoGround modes) // Reproduced from actual 1.8.9 client source @EventTarget public void onPacketSend(EventSendPacket eventSendPacket) { Packet packet = eventSendPacket.getPacket(); // OnGround mode: force onGround=true to prevent fall damage if (var3.equals("OnGround")) { if (packet instanceof C03PacketPlayer && mc.thePlayer.fallDistance > fallDistance.getValue()) { C03PacketPlayer c03 = (C03PacketPlayer) packet; c03.setOnGround(true); // tell server we are on ground // Also send an extra ground packet to be sure mc.thePlayer.sendQueue.addToSendQueueDirect( new C03PacketPlayer(true)); } } // NoGround mode: force onGround=false always (for bypass) if (var3.equals("NoGround")) { if (packet instanceof C03PacketPlayer) { C03PacketPlayer c03 = (C03PacketPlayer) packet; c03.setOnGround(false); } } // Stop mode: completely freeze fall packets if (var3.equals("Stop")) { if (packet instanceof C03PacketPlayer && mc.thePlayer.fallDistance > 3) { eventSendPacket.setCancelled(true); } } }
Vanilla hook: NetHandlerPlayServer.processPlayer() — the server-side handler that checks movement. EventSendPacket is the client-side hook. The key field is "onGround" (boolean in the constructor).
The onGround flag is the most-checked field by anti-cheats. NCP and AAC check that onGround matches the actual ground state. Verus checks for onGround=false spam. Watchdog checks C03 timing deltas.
Flight (Packet Mode / Position Spoof) (Gugustus, Mint B1, Koks v1, Tenacity 6.0, Sigma 4.11, November 0.2)

Allows the player to fly by spoofing position packets to bypass gravity checks.

There are many flight modes, but the most universal one is "Packet Flight". It relies on the fact that the server trusts the client"s position for small movements. The module intercepts S08PacketPlayerPosLook (server position correction) and cancels it. Meanwhile, it sends C06 packets with carefully modified Y values (+0.04, -0.04, etc) that hover in place. The server never sees the actual fall because S08 is cancelled. Some modes send C04 with massive Y offsets (+1000) to glitch the server into thinking the player is somewhere else entirely.

// Gugustus - Fly.java (Packet Flight) // Reproduced from actual 1.8.9 client source @EventTarget public void onPacketReceive(EventReceivePacket eventReceivePacket) { // Cancel S08 to prevent server position correction if (eventReceivePacket.getPacket() instanceof S08PacketPlayerPosLook) { eventReceivePacket.setCancelled(true); } } @EventTarget public void onPacketSend(EventSendPacket eventSendPacket) { // Spoof position with large offset to confuse anti-cheat if (eventSendPacket.getPacket() instanceof C03PacketPlayer) { C03PacketPlayer c03 = (C03PacketPlayer) eventSendPacket.getPacket(); c03.setX(mc.thePlayer.posX + 1000); c03.setZ(mc.thePlayer.posZ + 1000); c03.setOnGround(false); } } void doFlight() { // Send C06 to maintain altitude mc.thePlayer.sendQueue.addToSendQueueDirect( new C03PacketPlayer.C06PacketPlayerPosLook( mc.thePlayer.posX, mc.thePlayer.posY - 2.0, mc.thePlayer.posZ, mc.thePlayer.rotationYaw, mc.thePlayer.rotationPitch, false)); }
Vanilla hook: NetHandlerPlayServer.processPlayer() receives all C03/C04/C05/C06 events. S08PacketPlayerPosLook is sent by the server when it detects the player is out of position. Cancelling S08 is the core of packet flight.
NCP: Packet flight relies on cancelling S08 raw — patched with forced teleport confirm. AAC: Detects abnormal Y deltas — micro-offset flight (0.06, 0.016) bypasses this. Hypixel Watchdog: Detects any S08 cancel within 500ms — burst methods work briefly. Verus: Detects onGround=false spam. Most modern bypasses use a combination of C06 spoof + blink + timer adjustments.
Speed (Ground Spoof / Position Manipulation) (Gugustus, Koks v1, November 0.2, Sigma 3.9)

Makes the player move faster than normal by sending C04 packets at specific intervals with onGround=true.

The speed module sends C04PacketPlayerPosition at key moments with onGround=true (even when the player is in the air). This tricks the server"s movement calculation into giving the player extra velocity. The classic technique is "hop" mode: send ground-true position right as you jump, then let momentum carry you. Each hop sends 2-3 C04 packets with descending Y offsets. Some speed modes also intercept S08 to prevent rubberbanding.

// Gugustus - Speed.java — Ground spoof mc.thePlayer.sendQueue.addToSendQueue( new C04PacketPlayerPosition(mc.thePlayer.posX, someY, mc.thePlayer.posZ, true)); // Sigma 3.9 - Speed — C03 onGround true if (ep.isIncoming() && ep.getPacket() instanceof C03PacketPlayer) { // Read onGround for speed stage tracking } // S08 cancel to prevent rubberband (combined with Speed): if (event.getPacket() instanceof S08PacketPlayerPosLook && speedEnabled) { event.setCancelled(true); }
Vanilla hook: NetHandlerPlayServer.processPlayer() — the onGround flag significantly changes the server"s movement handling. The server uses onGround to decide whether to apply friction, air resistance, and fall damage.
The onGround flag is the most-checked field by anti-cheats. NCP and AAC check that onGround matches the actual ground state. Verus checks for onGround=false spam. Watchdog checks C03 timing deltas.
Blink (Packet Buffer + Release) (Gugustus, Koks v1, Tenacity 6.0, Swift Developer Alpha)

Buffers all movement packets (C03, C0B, C02) and releases them all at once, causing the player to teleport to their real location after they disable the module.

When enabled, the module intercepts and cancels C03PacketPlayer (and optionally C02/C0B) in EventSendPacket. The packets are stored in a List. The player still moves client-side but the server never sees any movement — so the player appears frozen to other players. When disabled, ALL buffered packets are sent at once to the server. The server processes them in sequence and the player teleports to their real position. This is also used for "backtrack" — releasing packets selectively to extend reach.

// Gugustus - Blink.java (packet buffer + release) // Reproduced from actual 1.8.9 client source private ArrayList<Packet> packets = new ArrayList<>(); @EventTarget public void onPacketSend(EventSendPacket event) { if (event.getPacket() instanceof C03PacketPlayer || event.getPacket() instanceof C02PacketUseEntity || event.getPacket() instanceof C0BPacketEntityAction) { packets.add(event.getPacket()); // store event.setCancelled(true); // cancel send } } @Override public void onDisable() { // Flush all held packets to server at once // The server processes them in order, teleporting the player for (Packet p : packets) { mc.thePlayer.sendQueue.addToSendQueue(p); } packets.clear(); }
Vanilla hook: EventSendPacket — buffer C03/C02/C0B. The packets must be released IN ORDER to maintain movement validity.
The onGround flag is the most-checked field by anti-cheats. NCP and AAC check that onGround matches the actual ground state. Verus checks for onGround=false spam. Watchdog checks C03 timing deltas.
Damage / Fall Damage Exploit (Koks v1, Jigsaw 0.26, November 0.2)

Deals self-damage (usually half a heart) to trigger certain game mechanics — crystal damage reduction, NoFall bypass, or pearl landing prep.

The module sends C04PacketPlayerPosition with a massive Y offset (e.g., +5000) and onGround=true. The server calculates fall distance = 5000 blocks, applies fall damage, subtracts from health, and sets the player back to the original position. The net effect is the player takes 1/2 heart of fall damage but stays in place.

// Koks v1 - Damage.java mc.thePlayer.sendQueue.addToSendQueue( new C04PacketPlayerPosition(mc.thePlayer.posX, mc.thePlayer.posY + 5000, mc.thePlayer.posZ, false)); // Jigsaw 0.26 - CommandDamage.java mc.thePlayer.sendQueue.addToSendQueue( new C04PacketPlayerPosition(mc.thePlayer.posX, mc.thePlayer.posY + 0.1, mc.thePlayer.posZ, false)); mc.thePlayer.sendQueue.addToSendQueue( new C04PacketPlayerPosition(mc.thePlayer.posX, mc.thePlayer.posY + 5000, mc.thePlayer.posZ, false)); // November 0.2 - Damage.java // Teleport up and send ground packet to trigger fall damage mc.thePlayer.sendQueue.addToSendQueue(new C04PacketPlayerPosition(x, y, z, true));
Vanilla hook: EventSendPacket — the C04 with +5000 Y triggers the server"s fall damage calculation when the next legitimate position shows the player on ground.
The onGround flag is the most-checked field by anti-cheats. NCP and AAC check that onGround matches the actual ground state. Verus checks for onGround=false spam. Watchdog checks C03 timing deltas.
Step (Staged Position Packets) (Gugustus)

Allows the player to walk up full blocks without jumping (normally only possible with slabs/stairs).

When the player walks into a block, the module fires a series of C04PacketPlayerPosition packets at specific Y offsets (0.42, 0.753, 1.0, 1.166, 1.0, 0.6). These correspond to the step animation that the server expects. The server accepts the movement because it receives position updates at each stage of the "step" instead of a single large Y jump.

// Gugustus - Step.java public void step() { double[] offsets = {0.42, 0.753, 1.0, 1.166, 1.0, 0.6}; for (double offset : offsets) { mc.thePlayer.sendQueue.addToSendQueue( new C04PacketPlayerPosition(x, y + offset, z, false)); } } // The module detects when the player is inside a block and triggers step
Vanilla hook: NetHandlerPlayServer.processPlayer(). The server accepts staged Y offsets if they match the step animation pattern.
The onGround flag is the most-checked field by anti-cheats. NCP and AAC check that onGround matches the actual ground state. Verus checks for onGround=false spam. Watchdog checks C03 timing deltas.
All clientbound movement intercepts use EventSendPacket with instanceof checks against C03PacketPlayer, C04PacketPlayerPosition, C05PacketPlayerLook, and C06PacketPlayerPosLook. These are the four IDs that map to the same MCP class hierarchy (C03PacketPlayer is abstract parent, C04/C05/C06 are concrete subclasses). The server-side handler is NetHandlerPlayServer.processPlayer(). On the client, these packets are generated in EntityPlayerSP.onUpdateWalkingPlayer() which runs every tick. Many modules hook onUpdateWalkingPlayer() directly to modify the packet BEFORE it"s sent, giving them more control than EventSendPacket (which fires after the packet is created). Vanilla send loop: EntityPlayerSP.onUpdateWalkingPlayer() → sets posX/posY/posZ/onGround → calls sendQueue.addToSendQueue(new C06PacketPlayerPosLook(...)) → EventSendPacket → NetworkManager.sendPacket() → Server.
