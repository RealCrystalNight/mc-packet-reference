Responds to C00PacketServerQuery with the server's status as a JSON string (MOTD, players, version, favicon, mod info).

Notes

The JSON response includes: version (name + protocol number), players (max + online + optional sample list), description (MOTD ChatComponent), and optionally favicon (base64-encoded PNG with 'data:image/png;base64,' prefix). For modded servers, may include Forge mod info.

implementation Implementation Cases

S00PacketServerInfo delivers the server's status JSON to the client during the STATUS handshake. This is the packet that populates the multiplayer server list with MOTD, player count, version, and favicon. The response is sent in response to C00PacketServerQuery. The JSON is parsed by OldServerPinger.java which extracts each field and populates the ServerData object. Modules that target this packet are rare but include: ServerScanner (parses version info to categorize servers), MOTD Spoofer (modifies the displayed MOTD client-side), and BrandDetector (reads Forge mod list from the JSON to identify server software).

Server List Pinger (Response Handler) (Rise 6.0, Jigsaw 0.26, RektSky B5, Sigma 4.11, Aqua 13, Memeware 7.3)

The vanilla handler that parses S00PacketServerInfo JSON to display server information in the multiplayer screen. Present in every single Minecraft client.

OldServerPinger.java creates an anonymous INetHandlerStatusClient with a handleServerInfo() method. This method parses the ServerStatusResponse JSON, extracts the MOTD (getServerDescription()), player count (getPlayerCountData()), version (getProtocolVersionInfo()), and favicon (getFavicon()). The parsed data populates the ServerData object which is then rendered in the GuiMultiplayer server list.

BrandDetector / ServerScanner (Rise 6.0, Sigma 4.11)

Parse the S00PacketServerInfo JSON to identify the server software (Spigot, Paper, BungeeCord, Forge, etc.) from version strings, player sample data, and Forge mod list (if present).

The module intercepts the S00PacketServerInfo before it reaches the vanilla handler, extracts the JSON string, and looks for identifying markers. For example: 'Paper' in the version name, 'BungeeCord' IP forwarding headers in the player sample, Forge mod list in the ForgeData section, or custom MOTD prefixes added by plugins. This information is used to auto-select appropriate disabler bypass modes and to display server info in HUD elements.

Vanilla hook: EventReceivePacket / PacketReceiveEvent — fires when S00PacketServerInfo arrives. The module reads the JSON before (or alongside) the vanilla handler.
Some servers use custom server software that returns misleading version strings to confuse server scanners. For example, a Paper server might report 'Spigot 1.8.8' instead of 'Paper 1.8.8'.
Client-side: OldServerPinger.handleServerInfo(S00PacketServerInfo). The JSON is deserialized with Gson and parsed to extract MOTD, player count, version, and favicon. After parsing, the client immediately sends C01PacketPing. Server-side: Sent by NetHandlerStatusServer in response to C00PacketServerQuery. The server builds a ServerStatusResponse from server.properties (MOTD, max players), the current player list (online count + sample), and server-icon.png (favicon). On Forge servers, additional mod info is appended. The STATUS handshake full sequence: C00Handshake(state=STATUS) → C00PacketServerQuery → S00PacketServerInfo → C01PacketPing → S01PacketPong → close.
