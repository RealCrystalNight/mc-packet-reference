C01PacketChatMessage is the client→server chat/command packet. Every chat message typed and every /command sends one. It carries a single String field (the message, max 100 chars). This packet is arguably the most-intercepted serverbound packet because it sits at the intersection of chat automation (AutoGG, Spammer), chat manipulation (Bypass, NameProtect), and client command systems (.bind, .toggle etc). Intercepting it lets a client silently consume commands without the server ever seeing them.

Command System (prefix-based) (Flux B39.11, Nekoware v1, Memeware 7.3, Gugustus, Jigsaw 0.26)

Allows the user to run client commands using a prefix like . or $ without sending them to the server.

In EventSendPacket, when the intercepted packet is C01PacketChatMessage, the module reads the message. If the message starts with the command prefix (e.g., "."), the event is cancelled (server never sees it) and the rest of the message is processed as a client-side command (toggle module, change setting, etc).

// Nekoware v1 - Commands.java if (p instanceof C01PacketChatMessage) { C01PacketChatMessage packet = (C01PacketChatMessage) p; String message = packet.getMessage(); if (message.startsWith(".")) { event.setCancelled(true); processCommand(message.substring(1)); // handle client-side return; } } // Memeware 7.3 - Commands.java (identical pattern) if (p instanceof C01PacketChatMessage) { String msg = ((C01PacketChatMessage) p).getMessage(); if (msg.startsWith(".")) { event.setCancelled(true); parseCommand(msg); } }
Vanilla hook: NetHandlerPlayClient.sendChatMessage() — this is where the client constructs a C01 and calls sendPacket. But for interception, EventSendPacket (fired inside NetworkManager.sendPacket()) is the correct hook. Most clients have a custom sendChatMessage override too.
There"s no anti-cheat check on this packet — the server just forwards your message to chat. However, servers can detect if you"re running client commands by noticing that messages starting with "still arrive at the server as empty strings if you don"t cancel properly.
AutoGG / AutoPlay / AutoRegister (Sigma 3.9, Astro 2.2.1, Tenacity 5.1, Helium B41420)

Automatically sends predefined chat messages when certain trigger events happen — "gg" after a kill, "/play mode" after game end, "/register password" on first join, "/tpaccept" on teleport request.

These modules listen for S02PacketChat (incoming). When the incoming message matches a trigger pattern (contains "Winner: ", "1st Killer - ", "You have been teleported", etc.), the module schedules or immediately sends a C01PacketChatMessage with the appropriate response. Some use a small delay to appear human. The sending is done via mc.thePlayer.sendChatMessage() which internally constructs and sends C01.

// Sigma 3.9 - AutoGG.java @EventTarget public void onReceive(EventReceivePacket ep) { if (ep.isIncoming() && ep.getPacket() instanceof S02PacketChat) { S02PacketChat s02 = (S02PacketChat) ep.getPacket(); String msg = s02.func_148915_c().getUnformattedText(); if (msg.contains("1st Killer - ") || msg.contains("Winner: ")) { mc.thePlayer.sendChatMessage("gg"); // internally creates C01 mc.thePlayer.sendChatMessage("/play solo_insane"); // AutoPlay } } } // Tenacity 5.1 - TPAKiller.java @Subscribe public void onEvent(EventReceivePacket event) { if (event.getPacket() instanceof S02PacketChat) { S02PacketChat s02 = (S02PacketChat) event.getPacket(); String msg = s02.getChatComponent().getUnformattedText(); if (msg.contains("has requested to teleport to you")) { mc.thePlayer.sendChatMessage("/tpaccept"); // sends C01 instantly } } }
Vanilla hook: EntityPlayerSP.sendChatMessage(String) — this is the vanilla method that constructs and sends C01PacketChatMessage. It calls sendQueue.addToSendQueue(new C01PacketChatMessage(msg)). Many modules either call this method or construct C01 directly via mc.getNetHandler().getNetworkManager().sendPacket(new C01PacketChatMessage(...)).
No anti-cheat blocks chat messages — this is purely a game feature. However, some servers limit message rate (e.g., Hypixel"s "You are sending messages too fast"). Smart AutoGG modules add a 500-1500ms random delay after detecting the trigger before sending "gg" to avoid spam filters.
Spammer (Sigma 3.9, Jigsaw 0.26, Mint B1, November 0.2)

Repeatedly sends a configurable message (or cycles through a list) at a set interval.

A timer runs in the background. When the timer reaches the configured delay (e.g., every 3 seconds), the module sends a C01PacketChatMessage with the next message from its queue. Some add a random suffix or number to avoid spam filters. The send uses either mc.thePlayer.sendChatMessage() or direct packet construction.

// Jigsaw 0.26 - ChatSpam.java int currentSpam = 0; public void onUpdate() { if (timer.delay(3000)) { // every 3 seconds mc.thePlayer.sendQueue.addToSendQueue(new C01PacketChatMessage(spamMessages[currentSpam])); currentSpam = (currentSpam + 1) % spamMessages.length; timer.reset(); } } // Mint B1 - Spammer.java mc.getNetHandler().addToSendQueueWithoutEvent(new C01PacketChatMessage(spam.get(thru))); // Uses addToSendQueueWithoutEvent to avoid re-triggering the event system (infinite loop prevention)
Vanilla hook: EntityPlayerSP.sendChatMessage(String) or direct sendQueue.addToSendQueue(new C01PacketChatMessage(...)). Both work.
Spam filters on servers detect identical messages repeating. Modules bypass this by: (a) adding a random suffix every message, (b) cycling through a message list, (c) randomizing the delay between 2000-4000ms instead of a fixed interval.
ChatBypass / AntiVoid (message modification) (Flux B39.11, Helium B41420)

Modifies outgoing chat messages to bypass anti-swear filters on servers. Also used to send messages that contain symbols the server blocks (like § color codes).

In EventSendPacket, if the packet is C01PacketChatMessage, the module cancels the original and creates a NEW C01PacketChatMessage with the message modified. Modifications include: adding special characters between letters (l→l), replacing letters with lookalikes (a→ɑ), inserting zero-width spaces, or splitting words with color codes. The cancel+re-send pattern is crucial — you can"t modify the String field in-place because the server already saw the original.

// Flux B39.11 - ChatBypass.java if (event.getPacket() instanceof C01PacketChatMessage) { C01PacketChatMessage packet = (C01PacketChatMessage) event.getPacket(); String msg = packet.getMessage(); // Don"t bypass commands if (!msg.startsWith("/")) { event.setCancelled(true); // cancel original String bypassed = bypass(msg); // apply bypass transformations mc.getNetHandler().getNetworkManager().sendPacketNoEvent(new C01PacketChatMessage(bypassed)); } } private String bypass(String s) { // Example: replace letters with lookalikes return s.replace("a", "ɑ").replace("e", "ɛ"); }
Vanilla hook: EntityPlayerSP.sendChatMessage() is the vanilla call site. EventSendPacket via NetworkManager.sendPacket() is the hook point.
Servers with advanced chat filters (like Sentinel) detect common bypass patterns (e.g., repeating characters, lookalike Unicode). Some clients have multiple bypass modes and auto-detect which one works.
Vanilla send path: EntityPlayerSP.sendChatMessage(String msg) → NetHandlerPlayClient.sendChatMessage(msg) → PacketUtil.createChatPacket(msg) → NetworkManager.sendPacket(new C01PacketChatMessage(msg)) → EventSendPacket fires → Netty writes → Server receives. Hook points: 1. EntityPlayerSP.sendChatMessage() — Override method to intercept all chat before packet creation. 2. EventSendPacket / EventReceivePacket — Check instanceof C01PacketChatMessage. This is what 90% of clients do. 3. NetworkManager.sendPacket() — A global hook that sees EVERY packet including C01. Useful for modules that need to filter packets before they reach specific handlers.
Flux B39.11: Has a full ChatBypass + AutoGG module. Intercepts outgoing C01 and modifies message before re-sending. Sigma 3.9: AutoGG + AutoTPA. Listens for S02 then sends C01 response. Nekoware v1: Command system processes "." prefix before server sees the message. Mint B1: Spammer uses addToSendQueueWithoutEvent to avoid re-triggering events. Jigsaw 0.26: Centralized onMessageSent(C01PacketChatMessage) handler in the main client class. Gugustus: AntiFlag module intercepts C01 and checks message content for flagged keywords. November 0.2: Disabler mode that blocks ALL outgoing C01 packets (anti-chat-report bypass).
