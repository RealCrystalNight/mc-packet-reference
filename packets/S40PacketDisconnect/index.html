Notes

This is the PLAY-state disconnect (vs S00PacketDisconnect for LOGIN). When received, the client immediately closes the network connection and displays the disconnect screen with the reason.

implementation Implementation Cases

S40PacketDisconnect kicks the player from the server during PLAY state. It carries a single IChatComponent (JSON chat) reason message. This packet is critical for AutoReconnect modules (detecting kicks and reconnecting), GhostMode/AntiKick modules (cancelling the disconnect to stay on the server after being kicked — a ghost state where the server has released your resources but you can still observe), and KickReasonDetector modules (parsing the reason to determine if it was an anti-cheat ban, a server restart, or a connection timeout).

AutoReconnect (Rise 6.0, Gugustus, Tenacity 5.1, Jigsaw 0.26, RektSky B5)

Detect when the player is kicked and automatically reconnect to the same server. Filters out ban-related kicks to avoid reconnecting to banned servers.

When S40PacketDisconnect arrives, the module reads the ChatComponent reason, cancels the packet, and starts a delayed reconnect. Before reconnecting, it checks if the reason contains 'banned', 'blacklisted', or 'permanently' — if so, it does NOT reconnect. For server restarts ('Server closed', 'Restarting'), it retries every 5 seconds. For anti-cheat kicks ('Kicked by Watchdog'), it toggles the Disabler mode and reconnects.

// Rise 6.0 - handling S40PacketDisconnect for AutoReconnect if (packet instanceof S40PacketDisconnect) { S40PacketDisconnect disconnect = (S40PacketDisconnect) packet; String reason = disconnect.getReason().getUnformattedText(); // Don't reconnect on bans if (reason.toLowerCase().contains("banned") || reason.toLowerCase().contains("blacklisted") || reason.toLowerCase().contains("permanently")) { return; // let vanilla handle the disconnect } // Cancel disconnect and reconnect event.setCancelled(true); // If anti-cheat kick, cycle disabler mode if (reason.toLowerCase().contains("watchdog") || reason.toLowerCase().contains("sentinel") || reason.toLowerCase().contains("aac")) { Disabler.cycleMode(); } // Reconnect after 3 second delay new Thread(() -> { try { Thread.sleep(3000); } catch (Exception e) {} Minecraft.getMinecraft().addScheduledTask(() -> { mc.displayGuiScreen(new GuiConnecting( mc.currentScreen != null ? mc.currentScreen : new GuiMainMenu(), mc, lastServerData)); }); }).start(); } // Vanilla S40PacketDisconnect definition: package net.minecraft.network.play.server; public class S40PacketDisconnect implements Packet<INetHandlerPlayClient> { private IChatComponent reason; public S40PacketDisconnect() {} public S40PacketDisconnect(IChatComponent reasonIn) { this.reason = reasonIn; } public void readPacketData(PacketBuffer buf) throws IOException { this.reason = buf.readChatComponent(); } public void processPacket(INetHandlerPlayClient handler) { handler.handleDisconnect(this); } public IChatComponent getReason() { return this.reason; } } // Jigsaw 0.26 - NetHandlerPlayClient handles disconnect: public void handleDisconnect(S40PacketDisconnect packetIn) { this.netManager.closeChannel(packetIn.getReason()); // Jigsaw hooks this for GhostMode } // RektSky B5 - NetHandlerPlayClient.java: public void handleDisconnect(S40PacketDisconnect packetIn) { this.netManager.closeChannel(packetIn.func_149603_c()); }
Vanilla hook: NetHandlerPlayClient.handleDisconnect(S40PacketDisconnect) — this method closes the NetworkManager channel, sets the game state to disconnected, and displays the disconnect screen GUI. Hacked clients fire events before this handler runs (PacketReceiveEvent, EventPacket.RECEIVE, etc.).
Cancelling S40 and reconnecting immediately is detectable: the server may still have the previous connection in its player list (name is taken) and the new connection will fail with 'You are already logged in'. GhostMode (cancelling S40 but NOT disconnecting) is detected by any anti-cheat that validates player online state against the player list — you'll be kicked again within seconds.
GhostMode / AntiKick (Jigsaw 0.26, Rise 6.0, RektSky B5)

Cancel the disconnect packet entirely to remain connected to the server in a 'ghost' state — the server thinks you're disconnected but your client is still receiving packets.

When S40PacketDisconnect arrives, the module cancels the event and prevents handleDisconnect from running. The NetworkManager channel stays open. The client continues to receive S packets and render the world, but the server has already removed the player from all lists (tab list, entity tracker, chunk senders). The player can observe but cannot interact. This is sometimes used for ghost spectating after death or to observe server behavior after being flagged.

// Jigsaw 0.26 - GhostMode.java intercepts disconnect: // In NetworkManager.channelRead0(), when S40 arrives: if (packet instanceof S40PacketDisconnect && GhostMode.isEnabled()) { // Cancel the packet — never reach handleDisconnect packet.cancelled = true; Jigsaw.chatMessage("GhostMode: Cancelled disconnect"); return; } // This relies on the fact that the server does not actively kick // via TCP close — it sends S40 first, then closes the connection. // If we cancel S40 and prevent the close, we stay connected. // Most server implementations send S40 and then close: // NetHandlerPlayServer.sendPacket(new S40PacketDisconnect(reason)); // player.connection.netManager.closeChannel(reason); // The second action (TCP close) can't be cancelled — ghost mode is temporary.
Vanilla hook: NetHandlerPlayClient.handleDisconnect(S40PacketDisconnect) — ghost mode prevents this method from running. However, the TCP close (NetworkManager.closeChannel) often follows immediately, which terminates the connection regardless.
GhostMode only works if the server sends S40 and waits before killing the TCP connection. Most servers immediately close the socket after S40, making ghost mode impossible. Some anti-cheats intentionally delay the TCP close to catch ghost-mode cheaters who then send invalid C03 packets after the disconnect.
Client-side: NetHandlerPlayClient.handleDisconnect(S40PacketDisconnect) — receives the kick reason and closes the connection. All major clients fire events here: - Jigsaw 0.26: Jigsaw.onPacketRecieved() in NetworkManager.channelRead0() - Rise 6.0: PacketReceiveEvent - RektSky B5: PacketReceiveEvent - Aqua 13: EventPacket.RECEIVE + EventReceivedPacket - Sigma 4.11: EventPacket with isIncoming() - Memeware 7.3: PacketInEvent Server-side: Sent by the server when kicking a player — NetHandlerPlayServer.kickPlayerFromServer() or ServerConfigurationManager.disconnect().
Rise 6.0: BlinkComponent exempts S40PacketDisconnect from the blink buffer (must be processed immediately). Jigsaw 0.26: GhostMode cancels S40 to stay connected. AutoReconnect uses the same packet for kick detection. RektSky B5: Disabler has a mode that cancels S40 on specific servers. Aqua 13: Backtrack exempts S40/S00 from its delay buffer. Gugustus: AntiKick + AutoReconnect both target this packet. Multiple GhostMode bypass variants. Tenacity 5.1: AutoReconnect reads kick reason and decides whether to reconnect.
