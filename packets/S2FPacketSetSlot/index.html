Notes

windowId=-1 special case: updates the cursor item (the item held by the mouse cursor). windowId=0: updates player inventory. S2F is sent for EVERY slot change — armor equips, hotbar changes, cursor picks up/drops. Inventory modules track S2F to detect armor swaps and item pickups.

implementation Implementation Cases

S2FPacketSetSlot updates a single inventory slot. Three fields: windowId (-1=cursor, 0=player inv), slot index, and the ItemStack (null = clear). Used for cursor item tracking, armor swaps (slots 5-8), and hotbar updates. InventoryManager modules read S2F to detect when armor is equipped (slot 5=helmet through 8=boots), and AutoPot detects when potion items change in the hotbar. Disabler modules may cancel S2F to prevent the client from seeing inventory changes, causing inventory desync.

Inventory Manager / AutoArmor Tracking (Rise 6.0, Sigma 4.11, Tenacity 6.0, Gugustus)

Track inventory slot changes via S2FPacketSetSlot to detect item pickups, armor swaps, and cursor changes without polling. AutoArmor uses S2F to detect when new armor items enter the inventory.

On each S2F, read windowId, slot, and item. If windowId=0 (player inventory) and slot is an armor slot (5-8), the module checks if the new item is better than currently equipped. If windowId=-1, the cursor item changed — used to detect when items are being dragged. The module tracks these changes to trigger actions without waiting for the next tick update.

Disabler (S2F Cancel) (Gugustus, Tenacity 5.1)

Cancel S2FPacketSetSlot to prevent the client from seeing specific inventory changes. Used to hide item manipulation from anti-cheat trackers.

When the Disabler is in a mode that manipulates inventory, S2F packets for specific slots are cancelled. The client never sees the slot update, so it maintains its previous (possibly fake) inventory state.
