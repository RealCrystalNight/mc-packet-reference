0x03 S03PacketEnableCompression

Enables zlib compression for all subsequent packets. Sets the compression threshold — packets >= this size are compressed.

Clientbound (Server → Client) · LOGIN · net/minecraft/network//S03PacketEnableCompression.java

Notes

After this packet, all future packets start with a VarInt prefix indicating the uncompressed length. If length=0, the data is uncompressed (smaller than threshold). If length>0, the remaining data is zlib-compressed and must be inflated.

implementation Implementation Cases

S03PacketEnableCompression is sent after login success and before any PLAY-state packets. It sets the compression threshold for the NetworkManager. Once compression is enabled, every packet's framed header includes a VarInt uncompressed-length prefix. Packets smaller than the threshold are sent uncompressed (length=0 prefix); larger packets are zlib-compressed. A compression threshold of -1 means 'never compress'. This packet matters for packet interception/tampering — man-in-the-middle tools need to decompress/recompress, and modules that modify raw packet bytes (like Proxy/VPN modules) must handle the compression layer correctly.

Disabler (Compression Manipulation) (Aqua 13, Rise 6.0)

Modify or cancel the compression threshold to manipulate how the server communicates, potentially causing desync between client and server packet framing.

Some disabler modes detect when compression is enabled and respond by sending oversized packets that would trigger server-side compression errors. If the threshold is changed client-side but not server-side, the client will attempt to decompress uncompressed data (or vice versa), causing the server to kick with 'bad packet' errors — this is sometimes used to trigger anti-cheat false positives or to cause targeted lag spikes.

Packet Sniffer / MITM Preparation (Rise 6.0, RektSky B5)

Track when compression is enabled so packet sniffing/capturing tools can correctly decompress subsequent packets for analysis.

Packet sniffers that intercept raw Netty channel data need to know when compression kicks in because the packet format changes. Before S03: `[VarInt length][VarInt packetId][data]`. After S03: `[VarInt frameLength][VarInt uncompressedLength][compressed/uncompressed data]`. The sniffer module records the threshold value from this packet and adjusts its parsing accordingly.
