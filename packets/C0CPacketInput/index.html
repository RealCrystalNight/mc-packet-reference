Player steers a vehicle (boat or horse). Sends sideways/forward movement, jump flag, and unmount flag.

Notes

Sent every tick while riding a vehicle. Rise 6.0 uses C0CPacketInput in its Disabler module for three different modes: VehicleDisabler (send empty C0C on pre-motion), InputDisabler (send C0C with fix- ed values on strafe), and GhostlyDisabler (send C0C alongside C03 movement). The empty constructor C0CPacketInput() defaults to (0,0,false,false) which freezes the boat.

implementation Implementation Cases

C0CPacketInput drives vehicle movement client-side. Four fields: sideways/forward (-1 to 1 floats), jump (boolean — launches boat, charges horse), unmount (boolean — dismount). Rise 6.0 exploits this in three Disabler modes that send spoofed input values alongside movement packets to desync the player's vehicle state.

VehicleDisabler (Rise 6.0) (Rise 6.0)

Send empty C0CPacketInput every pre-motion to freeze the vehicle while the player moves independently via C03 packets. Creates a desync where the server thinks the player is sitting in a stationary boat but C03 shows movement.

On every PreMotionEvent, sends new C0CPacketInput() — an empty constructor that defaults to (0,0,false,false). This tells the server the vehicle has no input (sideways=0, forward=0, no jump, no unmount). Meanwhile, the module continues sending C03 packets that move the player. The server sees the boat frozen but the player moving — this can bypass vehicle-based anti-cheat checks.

InputDisabler (Rise 6.0) (Rise 6.0)

Send spoofed C0CPacketInput with maximum forward and no sideways on every strafe event. This is used to override legitimate vehicle input with values that desync anti-cheat tracking.

On StrafeEvent, sends C0CPacketInput(0.98f, 0, true, true) — near-max forward, jump=true, unmount=true. This overwrites whatever legitimate input the player is sending, creating a desync between the vehicle's actual movement and what C0C reports.

GhostlyDisabler (Rise 6.0) (Rise 6.0)

Combines C0CPacketInput spoofing with C03 movement manipulation for a comprehensive disabler mode. Sends C0C on every C03 send, and also rounds C03 Y to ground values.

When a C03PacketPlayer is detected, the module first sends C0CPacketInput() (empty — freezes vehicle), then rounds the C03 Y value to the nearest 0.015625 (ground) and forces onGround=true. This combines the vehicle input freeze with movement ground spoofing.
