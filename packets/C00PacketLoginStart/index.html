0x00 C00PacketLoginStart

Starts the login process by sending the player's GameProfile (username) to the server.

Notes

Sent immediately after C00Handshake (with state=LOGIN). The GameProfile has a username but the UUID is null — the server fills in the UUID from Mojang's auth servers and returns it in S02PacketLoginSuccess. This is the first step of the login handshake after the initial HANDSHAKING state.

implementation Implementation Cases

C00PacketLoginStart carries the player's username to the server to initiate the login process. It's sent from GuiConnecting.java (multiplayer), RealmsConnect.java (Realms), or Minecraft.java (singleplayer integrated server) immediately after the C00Handshake packet. The packet contains a GameProfile with only the username populated — the UUID is assigned later by Mojang's session servers. Modules target this packet for: NameSpoof (changing the username field before it reaches the server), OfflineUUID (setting a custom UUID for cracked/offline servers), and BungeeCord IP-Forwarding (injecting player IP/ UUID data into the handshake).

NameSpoof / Nick (Rise 6.0, Gugustus, Jigsaw 0.26, Sigma 4.11)

Modify the player's username before authentication to appear as a different player on the server. Works on offline/cracked servers that don't verify with Mojang.

The module intercepts C00PacketLoginStart before it's sent and replaces the GameProfile's name field with a custom string. On offline-mode servers, the server accepts whatever name is given and the player appears as that username. On online-mode servers, this will fail authentication because Mojang's session server will verify that the connecting player owns the username.

// Rise 6.0 - GuiConnecting.java sends login start: NetworkManager networkmanager = NetworkManager.createNetworkManagerAndConnect( new InetSocketAddress(ip, port), false); networkmanager.setNetHandler(new NetHandlerLoginClient(networkmanager, mc, guiScreen)); networkmanager.sendPacket(new C00Handshake(47, ip, port, EnumConnectionState.LOGIN)); networkmanager.sendPacket(new C00PacketLoginStart(mc.getSession().getProfile())); // Jigsaw 0.26 - GuiConnecting (same vanilla flow): networkManager.sendPacket(new C00Handshake(47, GuiConnecting.this.ip, GuiConnecting.this.port, EnumConnectionState.LOGIN)); networkManager.sendPacket(new C00PacketLoginStart(GuiConnecting.this.mc.getSession().getProfile())); // Vanilla C00PacketLoginStart definition: public class C00PacketLoginStart implements Packet<INetHandlerLoginServer> { private GameProfile profile; public C00PacketLoginStart() {} public C00PacketLoginStart(GameProfile profileIn) { this.profile = profileIn; } public void readPacketData(PacketBuffer data) throws IOException { this.profile = new GameProfile((UUID) null, data.readStringFromBuffer(16)); } public void writePacketData(PacketBuffer data) throws IOException { data.writeString(this.profile.getName()); // only sends the name! } public GameProfile getProfile() { return this.profile; } }
Vanilla hook: NetHandlerLoginServer.processLoginStart(C00PacketLoginStart) — the server-side handler that extracts the username, generates a UUID (online-mode: from Mojang; offline-mode: offlinePlayer UUID), and prepares encryption if online-mode is enabled. Rise 6.0 BlinkComponent exempts C00PacketLoginStart from blink: if (packet instanceof C00PacketLoginStart) return; // never delay login packets
Name spoofing only works on offline/cracked servers. Online-mode servers will reject the connection because the Mojang session server confirms the username. Some servers use LoginSecurity plugins that check for rapid name changes or mismatched UUIDs. BungeeCord servers pass the real IP through the handshake — name spoofing doesn't affect this.
OfflineUUID / Cracked Login (Jigsaw 0.26, Sigma 4.11)

Set a custom UUID before login for consistent identification on cracked/offline servers where UUIDs are normally regenerated each session.

On offline-mode servers, the server generates a UUID from the username using UUID.nameUUIDFromBytes(). To have a fixed UUID across sessions, modules intercept S02PacketLoginSuccess and replace the returned UUID with a predefined one, or pre-modify the GameProfile in C00PacketLoginStart to include a fixed UUID.

// Jigsaw 0.26 - NetHandlerLoginServer receives login start: public void processLoginStart(C00PacketLoginStart packetIn) { GameProfile gameprofile = packetIn.getProfile(); if (!this.server.isServerInOnlineMode()) { // Offline mode: generate UUID from username UUID uuid = UUID.nameUUIDFromBytes(("OfflinePlayer:" + gameprofile.getName()).getBytes(Charsets.UTF_8)); // Module can intercept S02LoginSuccess to replace this UUID } this.gameProfile = gameprofile; } // Vanilla - entity ID is always the player for login start // No Module in current source targets this directly // BlinkComponent.java (Rise 6.0) exempts it: // if (packet instanceof C00PacketLoginStart) return;
Vanilla hook: NetHandlerLoginServer.processLoginStart() on the server side. The server generates the UUID based on online/offline mode.
Anti-cheats typically don't inspect login packets for cheats — this is pre-gameplay. However, some anti-VPN/anti-proxy plugins inspect the connection timing between C00Handshake and C00PacketLoginStart to detect automated connections.
Client-side: GuiConnecting.java → NetworkManager.sendPacket(new C00PacketLoginStart(profile)). The sendPacket call fires events in all major clients: - Rise 6.0: PacketSendEvent → modules can intercept before send - Jigsaw 0.26: Jigsaw.onPacketSent() hook in NetworkManager.sendPacket() - RektSky B5: HLPacketSentEvent + PacketSentEvent - Aqua 13: EventPacket (fired from addToSendQueue) - Sigma 4.11: EventPacket with isOutgoing() - Memeware 7.3: PacketOutEvent fired from addToSendQueue() Server-side: NetHandlerLoginServer.processLoginStart() — validates name, generates UUID, starts encryption flow.
Rise 6.0: BlinkComponent + PingSpoofComponent both exempt C00PacketLoginStart from delay queues. Jigsaw 0.26: NetworkManager.sendPacket() runs Jigsaw.onPacketSent() before dispatching. Memeware 7.3: NetHandlerPlayClient.addToSendQueue() fires PacketOutEvent — C00PacketLoginStart passes through this same hook. Aqua 13: EventPacket + EventReceivedPacket fire in channelRead0()/addToSendQueue(). No specific modules target C00PacketLoginStart.
